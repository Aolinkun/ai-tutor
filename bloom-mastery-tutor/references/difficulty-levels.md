# 难度层级参考

## 通用认知层级（基于 Bloom 分类法）

| 层级 | 能力描述 | 典型问题形式 |
|------|----------|-------------|
| 1. 记忆 | 能回忆基本概念和术语 | "X 是什么？" |
| 2. 理解 | 能用自己的话解释 | "用你自己的话说说为什么..." |
| 3. 应用 | 能在新情境中使用概念 | "如果...，你会怎么用 X？" |
| 4. 分析 | 能拆解、比较、找关系 | "X 和 Y 有什么本质区别？" |
| 5. 评估 | 能批判性判断优劣 | "X 方法有什么局限？什么情况下不适用？" |
| 6. 创造 | 能综合产出新东西 | "如何用这些概念设计一个...？" |

## 难度推进策略

### 快速推进（学生表现强）
- 连续 2 轮完全正确 → 跳跃 2 个层级
- 回答有超出预期的深度 → 直接进入分析/评估层

### 原地巩固（学生有困难）
- 同一概念换角度（类比、反例、极端情况）
- 拆分成更小的子问题
- 先给出部分答案，让学生补全

### 回退（学生明显卡住）
- 回到上一层级的具体例子
- 建立直觉后再回到抽象定义
- 使用"想象你要向朋友解释..."类问题降低门槛

## 学科特定建议

### 哲学/人文
- 从经典文本的具体段落入手，而非抽象概念
- 用对比（这个思想家 vs 另一个）来建立坐标系
- 历史背景往往是理解思想的关键

### 数学/逻辑
- 先直觉，再形式化
- 一个概念配一个具体数字例子
- 让学生自己构造反例来检验理解

### 编程/技术
- 先跑通一个最小可用示例
- 让学生预测代码输出，再运行验证
- 错误信息是最好的学习材料

### 历史/社会科学
- 强调因果关系和多重视角
- "如果当时...，结果会怎样？"类反事实问题
- 连接到学生熟悉的当代现象
